# 数字人播报

本文档说明 ReAngle 如何将改写结果转为口播稿并生成 HeyGen 数字人视频；当前仅使用 HeyGen **Video Generation API**（`/v2/video/generate`），包括用户流程、后端接口、参数配置及前端交互。

本功能在「文章改写」流程之上，将改写结果一键转为数字人播报视频。用户在结果区域切换到 **Avatar** 标签，可先生成并编辑口播稿，再以口播稿调用 HeyGen 生成数字人视频，并在页面内预览与下载。

---

## 一、用户使用流程

用户完成改写后，切换到结果页中的 **Avatar** 标签。页面有两个主要按钮：「生成口播稿」从改写结果生成适合朗读的口播稿，显示在可编辑文本框中并带字数/词数统计与复制；「生成数字人播报」以当前口播稿文本调用 HeyGen 生成数字人视频。若尚未生成口播稿就点击「生成数字人播报」，系统会先自动生成口播稿再继续。生成完成后视频在口播稿下方预览，并提供「下载视频」按钮。为控制时长与费用，口播稿超过约 800 字时前端会拦截并提示先缩减字数。

---

## 二、系统架构与数据流

整体流程：**改写结果 → 口播稿 → 数字人视频**。

1. 用户在主应用完成多源输入与 ReAngle 生成后，得到 `summary` 与 `rewritten_content`（改写正文）。
2. 在 Avatar 标签中，前端将改写正文发送到 **`POST /api/v2/media/voiceover`**，由 `voiceover_client` 调用 OpenAI 生成口播稿，响应字段为 `script`，前端展示并允许编辑。
3. 用户确认口播稿后，前端将口播稿文本发送到 **`POST /api/v2/media/avatar`**，由 `avatar_client` 调用 HeyGen **`/v2/video/generate`** 生成数字人视频，返回 `video_url`；前端内嵌 `<video>` 预览，下载通过 **`GET /api/v2/media/avatar/download?url=...`** 代理实现。

## 三、后端实现概览

### 3.1 路由与接口（v2 Media）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v2/media/voiceover` | 接收 `text`（改写后文章），返回口播稿 `script`。 |
| POST | `/api/v2/media/avatar` | 接收 `text`（口播稿），调用 HeyGen 生成视频，返回 `video_url`。 |
| GET | `/api/v2/media/avatar/download` | 查询参数 `url` 为 HeyGen 视频 URL；白名单校验后代理下载，`Content-Disposition: attachment`。 |

口播稿生成建议控制在约 300–600 字，以对应 1–2.5 分钟视频与较好节奏。下载接口仅允许域名：`files2.heygen.ai`、`api.heygen.com`、`heygen.ai`，避免 SSRF。

### 3.2 数字人视频生成：HeyGen Video Generation API

后端由 **`app/services/media_outputs/avatar_client.py`** 中的 `AvatarClient` 封装 HeyGen **V2 Video Generation API**，不使用 Video Agent API。

**调用流程：**

1. 使用口播稿文本向 HeyGen **`POST /v2/video/generate`** 提交生成任务，请求体见下表。
2. 拿到 `video_id` 后，轮询 **`GET /v1/video_status.get?video_id=...`** 直至 `status` 为 `completed` 或 `failed`，取回 `video_url`。
3. 轮询策略：间隔 20 秒，最多 60 次（约 20 分钟），超时则返回错误提示。

**请求体结构（与代码一致）：**

- `caption`：是否开启字幕（由 `HEYGEN_CAPTION` 控制，默认 `false`）。
- `dimension`：`width`、`height`，来自 `HEYGEN_VIDEO_WIDTH`、`HEYGEN_VIDEO_HEIGHT`（默认 640×360）。
- `video_inputs`：单元素数组，每项包含：
  - **character**：`type: "avatar"`，`avatar_id`、`avatar_style`、`talking_style`；
  - **voice**：`type: "text"`，`input_text`（口播稿）、`voice_id`、`speed`、`pitch`、`duration`；
  - **background**：`type: "color"`，`value`、`play_style`、`fit`；
  - **text**：占位用（可留空），便于后续扩展字幕。

### 3.3 环境变量与参数配置

所有数字人相关参数通过环境变量配置，未设置时使用下列默认值。

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `HEYGEN_API_KEY` | — | **必填**，HeyGen API 密钥。 |
| `HEYGEN_AVATAR_ID` | `Fernando_sitting_businessindoor_front` | 数字人形象 ID。 |
| `HEYGEN_AVATAR_STYLE` | `normal` | 形象风格。 |
| `HEYGEN_TALKING_STYLE` | `stable` | 说话风格。 |
| `HEYGEN_VOICE_ID` | `bad86f2c05d843c3901e110fbddbe86a` | 声音 ID。 |
| `HEYGEN_VOICE_SPEED` | `1` | 语速。 |
| `HEYGEN_VOICE_PITCH` | `0` | 音高。 |
| `HEYGEN_VOICE_DURATION` | `1` | 时长相关。 |
| `HEYGEN_VIDEO_WIDTH` | `640` | 视频宽度。 |
| `HEYGEN_VIDEO_HEIGHT` | `360` | 视频高度。 |
| `HEYGEN_BACKGROUND_COLOR` | `#FFFFFF` | 背景颜色。 |
| `HEYGEN_BACKGROUND_PLAY_STYLE` | `freeze` | 背景播放方式。 |
| `HEYGEN_BACKGROUND_FIT` | `cover` | 背景填充方式。 |
| `HEYGEN_CAPTION` | `false` | 是否开启字幕（`true`/`false`）。 |

在根目录 `.env` 中按需覆盖以上变量即可更换主播、声音或分辨率，无需改代码。

## 四、前端实现概览

结果区域 Avatar 相关逻辑在 **`frontend/src/pages/MainApp.tsx`** 与 **`frontend/src/components/ReAngleView.tsx`** 中：

- **口播稿**：调用 `POST /api/v2/media/voiceover`，请求体 `{ text: result.rewritten_content }`，响应 `script` 填入可编辑框。
- **数字人视频**：调用 `POST /api/v2/media/avatar`，请求体 `{ text: 当前口播稿文本 }`，响应 `video_url` 用于 `<video src={...}>` 预览。
- **下载**：`GET /api/v2/media/avatar/download?url=${encodeURIComponent(avatarVideoUrl)}`，新窗口打开触发浏览器下载。

口播稿超过约 800 字时前端拦截并提示缩短后再生成。

---

## 五、费用与性能策略

当前仅使用 HeyGen **Video Generation API**（显式配置 avatar、声音、分辨率与背景），口播稿作为 `input_text` 传入，画面与时长可控，成本与延迟相对稳定。前端 800 字上限与后端轮询超时策略共同控制单次生成成本与体验。
