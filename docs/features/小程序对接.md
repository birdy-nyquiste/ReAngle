## 概述

睡前故事生成器 API 提供基于关键词的睡前故事生成服务。API 采用 RESTful 架构，支持两种响应模式：

1. **直接返回模式**（适用于云服务如 Render.com）：POST `/api/v1/miniprogram/generate` 直接返回完整故事内容
2. **两步流程模式**（适用于有文件系统权限的环境）：POST `/api/v1/miniprogram/generate` 返回 `resultUrl`，然后然后 GET 该 URL 获取结果 或 GET `/api/v1/miniprogram/results/{job_id}.json` 获取结果

所有响应均为 JSON 格式，使用 UTF-8 编码。

## API 端点

### 1. 健康检查

**端点**: `GET /api/v1/miniprogram/health`

**功能**: 检查服务是否正常运行

**请求示例**:

```bash
curl -X GET https://article-reangle.onrender.com/api/v1/miniprogram/health
```

**响应示例**:

```json
{
  "status": "healthy",
  "message": "睡前故事生成器后端服务运行正常",
  "timestamp": 1699123456.789
}
```

**状态码**: `200 OK`

---

### 2. 生成故事

**端点**: `POST /api/v1/miniprogram/generate`

**功能**: 根据提供的素材和需求生成睡前故事

**请求体**:

```json
{
  "text": "animals",
  "prompt": "toddler",
  "keywords": {
    "material": "animals",
    "requirement": "toddler"
  },
  "client": {
    "requestId": "a1b2c3d4"
  }
}
```

**字段说明**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `text` | string | 否 | 故事素材关键词（兼容字段，优先使用 `keywords.material`） |
| `prompt` | string | 否 | 故事需求关键词（兼容字段，优先使用 `keywords.requirement`） |
| `keywords` | object | 否 | 关键词对象 |
| `keywords.material` | string | 否 | 故事素材类型，见[关键词说明](#关键词说明) |
| `keywords.requirement` | string | 否 | 故事需求类型，见[关键词说明](#关键词说明) |
| `client` | object | 否 | 客户端信息 |
| `client.requestId` | string | 否 | 客户端请求ID，用于追踪请求 |

**响应模式 1: 直接返回结果**（文件系统不可写时）

**状态码**: `200 OK`

**响应体**:

```json
{
  "success": true,
  "rewritten_text": "从前有一只小兔子，它住在森林深处的一个温暖的小洞里...",
  "title": "小兔子的友谊故事",
  "length": "适中",
  "age": "3-5岁",
  "theme": "动物",
  "client": {
    "requestId": "a1b2c3d4"
  }
}
```

**响应模式 2: 返回 resultUrl**（文件系统可写时）

**状态码**: `200 OK`

**响应体**:

```json
{
  "success": true,
  "jobId": "b4f5a6c7-d8e9-4f10-a11b-12c13d14e15f",
  "resultUrl": "https://article-reangle.onrender.com/api/v1/miniprogram/results/b4f5a6c7-d8e9-4f10-a11b-12c13d14e15f.json"
}
```

**客户端处理逻辑**:

1. 检查响应中是否存在 `rewritten_text` 字段  
   - 如果存在，直接使用该数据（响应模式 1）  
2. 如果不存在 `rewritten_text`，检查是否存在 `resultUrl` 字段  
   - 如果存在，使用 GET 请求访问 `resultUrl` 获取结果（响应模式 2）

**状态码**: `200 OK`

## 错误处理

### 错误响应格式

```json
{
  "success": false,
  "error": "服务器内部错误: 具体错误信息",
  "error_type": "ErrorType"
}
```

### HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| `200` | 请求成功 |
| `400` | 请求参数错误 |
| `404` | 资源不存在（如 jobId 对应的结果文件不存在） |
| `500` | 服务器内部错误 |

### 常见错误场景

#### 1. 服务器内部错误 (500)

**响应**:

```json
{
  "success": false,
  "error": "服务器内部错误: 具体错误描述",
  "error_type": "ValueError"
}
```

**可能原因**:

- 后端代码执行异常
- 文件系统权限问题（已自动处理）
- 依赖包缺失

**处理建议**:

- 检查服务器日志
- 确认后端服务正常运行
- 验证请求数据格式

#### 2. 结果文件不存在 (404)

**响应**:

```http
HTTP/1.1 404 Not Found
```

**可能原因**:

- `jobId` 不存在或已过期
- 文件系统清理了临时文件

**处理建议**:

- 重新调用 `/api/v1/miniprogram/generate` 接口
- 检查 `resultUrl` 是否正确

## 关键词说明

### 故事素材 (material)

用于指定故事的主题和背景：

| 关键词 | 说明 | 示例故事 |
|--------|------|----------|
| `free` | 自由创作，由AI自由发挥 | 小精灵的梦境工厂 |
| `animals` | 小动物主题 | 小兔子的友谊故事 |
| `fairy` | 童话世界 | 小公主的星空愿望 |
| `nature` | 自然探索 | 小狐狸的夜游奇遇 |
| `friendship` | 友谊与分享 | 友谊的种子 |
| `courage` | 勇气与成长 | 小老鼠的勇气 |

### 故事需求 (requirement)

用于指定故事的目标受众和风格：

| 关键词 | 说明 | 适合年龄 |
|--------|------|----------|
| `toddler` | 3-5岁幼儿，简单易懂，语言温柔 | 3-5岁 |
| `child` | 6-8岁儿童，情节丰富，寓教于乐 | 6-8岁 |
| `sleepy` | 温柔催眠，节奏舒缓，有助于入睡 | 全年龄 |
| `adventure` | 冒险故事，充满想象力 | 6-12岁 |
| `educational` | 教育意义，包含生活道理和价值观 | 4-10岁 |

### 关键词组合示例

```json
{
  "keywords": {
    "material": "animals",
    "requirement": "toddler"
  }
}
```

这将生成一个适合 3-5 岁幼儿的动物主题睡前故事。

## 代码示例

### JavaScript / 微信小程序

```javascript
// 生成故事
function generateStory(material, requirement) {
  return new Promise((resolve, reject) => {
    wx.request({
      url: 'https://article-reangle.onrender.com/api/v1/miniprogram/generate',
      method: 'POST',
      header: {
        'Content-Type': 'application/json'
      },
      data: {
        keywords: {
          material: material,
          requirement: requirement
        },
        client: {
          requestId: String(Date.now())
        }
      },
      success: (res) => {
        if (res.statusCode === 200) {
          // 检查是否直接返回了结果
          if (res.data.rewritten_text) {
            resolve(res.data);
          }
          // 或者需要二次请求
          else if (res.data.resultUrl) {
            wx.request({
              url: res.data.resultUrl,
              method: 'GET',
              success: (resultRes) => {
                if (resultRes.statusCode === 200) {
                  resolve(resultRes.data);
                } else {
                  reject(new Error('获取结果失败'));
                }
              },
              fail: reject
            });
          } else {
            reject(new Error('响应格式错误'));
          }
        } else {
          reject(new Error(res.data?.error || '请求失败'));
        }
      },
      fail: reject
    });
  });
}

// 使用示例
generateStory('animals', 'toddler')

    console.log('故事内容:', result.rewritten_text);
```

### Python

```python
import requests
import json

def generate_story(material, requirement, base_url='https://article-reangle.onrender.com/api/v1/miniprogram'):
    """
    生成睡前故事
    
    Args:
        material: 故事素材类型
        requirement: 故事需求类型
        base_url: API基础URL
    
    Returns:
        dict: 包含故事内容的字典
    """
    url = f"{base_url}/generate"
    
    payload = {
        "keywords": {
            "material": material,
            "requirement": requirement
        },
        "client": {
            "requestId": f"req_{int(time.time())}"
        }
    }
    
    headers = {
        "Content-Type": "application/json"
    }
    
    try:
        # 第一步：POST /generate
        response = requests.post(url, json=payload, headers=headers)
        response.raise_for_status()
        data = response.json()
        
        # 检查是否直接返回了结果
        if 'rewritten_text' in data:
            return data
        
        # 如果需要二次请求
        if 'resultUrl' in data:
            result_url = data['resultUrl']
            result_response = requests.get(result_url)
            result_response.raise_for_status()
            return result_response.json()
        
        raise ValueError("响应格式错误：缺少 rewritten_text 或 resultUrl")
        
    except requests.exceptions.RequestException as e:
        raise Exception(f"请求失败: {str(e)}")

# 使用示例
if __name__ == '__main__':
    import time
    
    try:
        result = generate_story('animals', 'toddler')
        print(f"故事标题: {result['title']}")
        print(f"故事内容: {result['rewritten_text']}")
        print(f"适合年龄: {result['age']}")
        print(f"故事主题: {result['theme']}")
    except Exception as e:
        print(f"错误: {e}")
```

### 2. 错误重试机制

对于网络错误，建议实现指数退避重试：

```javascript
async function generateStoryWithRetry(material, requirement, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await generateStory(material, requirement);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      await new Promise(resolve => setTimeout(resolve, Math.pow(2, i) * 1000));
    }
  }
}
```

```javascript
function validateParams(material, requirement) {
  const validMaterials = ['free', 'animals', 'fairy', 'nature', 'friendship', 'courage'];
  const validRequirements = ['toddler', 'child', 'sleepy', 'adventure', 'educational'];
```

## 常见问题

### Q1: 为什么有时候直接返回结果，有时候返回 resultUrl？

**A**: 这取决于后端服务器的文件系统权限：

- **直接返回结果**：服务器文件系统不可写（如 Render.com 免费版），直接返回完整故事内容
- **返回 resultUrl**：服务器有文件系统权限，先生成文件，再返回文件URL

客户端应该同时支持两种模式，优先检查 `rewritten_text`，如果没有则使用 `resultUrl`。

### Q2: 请求失败返回 500 错误怎么办？

**A**:

1. 检查服务器日志，查看具体错误信息
2. 验证请求参数格式是否正确
3. 确认服务器依赖包已正确安装（Flask、flask-cors）
4. 检查服务器资源是否充足（内存、磁盘空间）

### Q3: resultUrl 访问返回 404？

**A**:

1. 检查 `jobId` 是否正确
2. 文件可能已被清理（临时文件有生命周期）
3. 重新调用 `/api/v1/miniprogram/generate` 接口生成新故事

### Q4: 如何选择合适的素材和需求组合？

**A**:

- **3-5岁幼儿**：`animals` + `toddler` 或 `fairy` + `toddler`
- **6-8岁儿童**：`nature` + `child` 或 `friendship` + `child`
- **睡前催眠**：任意素材 + `sleepy`
- **教育意义**：`friendship` + `educational` 或 `courage` + `educational`

### Q5: 故事内容可以自定义吗？

**A**: 当前版本使用预定义的故事模板，根据关键词组合返回对应的故事。如需自定义故事内容，需要修改后端代码的 `generate_story()` 函数。

### Q6: API 有调用频率限制吗？

**A**: 当前版本没有设置调用频率限制，但建议：

- 避免在短时间内发送大量请求
- 实现客户端请求队列
- 考虑添加请求缓存机制

### Q7: 支持并发请求吗？

**A**: 支持。每个请求都会生成唯一的 `jobId`，不会相互干扰。
