# 多源输入队列处理

本文档说明 ReAngle「多源输入队列」的整体设计：用户将多源内容（纯文本、文件、URL 等）入队并提交后，后端统一抽取与聚合为 `clean_text` 写入 Session，供后续 DeAngle / ReAngle 使用；当前 API 为 **`POST /api/v2/inputs/`**。

---

## 一、用户视角的使用方式

在主应用页面（`MainApp`）左侧，用户可以通过四种方式添加内容：

1. **Text**：直接粘贴或输入纯文本片段。  
2. **File**：上传 `.docx`、`.pdf`等文件。  
3. **URL**：粘贴网页链接，由系统抓取正文。  
4. **YouTube**：粘贴视频链接，当前按链接文本纳入上下文。  

每次在对应 Tab 中填写好内容后点击「Add to Queue」，该输入会作为一条独立记录加入队列列表（右侧显示简要信息，例如标题/文件名、字符数等）。用户可以多次添加、多种类型混合添加，也可以从队列中删除不需要的条目。  

当队列准备完毕并提交时，前端将队列序列化后调用 **`POST /api/v2/inputs/`**，后端负责多源抽取与聚合，结果写入 Session；用户随后可执行 DeAngle、再执行 ReAngle，得到摘要与改写正文。

---

## 二、前端实现概要

前端的输入队列逻辑主要集中在 `frontend/src/pages/MainApp.tsx`：

- 使用 `inputItems: InputItem[]` 维护当前队列，每个元素包含 `id`、`type`（`text` / `file` / `url` / `youtube`）、`content` 以及 `meta`（标题、摘要信息）。
- 不同 Tab 对应不同输入方式：
  - 文本与 URL 直接将字符串写入 `content`；
  - 文件类型会在前端只保存 `File` 对象引用，真正上传在提交时处理。
- 点击「Add to Queue」时，根据当前 Tab 和输入内容构造 `InputItem` 并追加到队列，同时清空该输入框。
- 在点击「Transform Content」时：
  - 创建 `FormData` 并遍历 `inputItems`：
    - 对于文件：将 `File` 附加到 `FormData`，记录 `contentKey`；
    - 对于其他类型：直接将文本内容放到序列化对象中。
  - 将上述对象数组序列化为 JSON 字符串，放入表单字段 `inputs` 中一并提交给后端。

这样设计的好处是：前端只需维护一个统一的队列结构，不同来源的输入在「添加」和「提交」两个阶段被规范化，后端拿到 `inputs` 后可以用同一套逻辑进行拆解和抽取。

---

## 三、后端抽取与聚合流程

后端逻辑在 **`app/routers/v2/inputs.py`** 的 **`POST /api/v2/inputs/`** 与 **`app/services/inputs`**（如 `inputs_service.process_inputs`）中：

1. **解析队列**：从表单字段 `inputs` 读取 JSON（`List[InputItem]`），结合上传的 `files`；解析失败时抛出 `InvalidInputError`。
2. **按类型抽取内容**：遍历条目，根据 `type` 分派——`text` 直接取 `content`；`url` 抓取正文；`file` 按后缀用对应抽取器（PDF/DOCX/图片等）；抽取结果可加来源标记。
3. **内容聚合**：将各条结果拼接为一段 `clean_text`，若为空则报错。
4. **写入 Session**：`clean_text` 存入当前会话，供后续 **`POST /api/v2/deangle/`** 与 **`POST /api/v2/reangle/`** 使用；改写与摘要由 ReAngle 接口根据用户设置中的模型与 prompt 单独完成，不在此接口返回。

---

## 四、设计动机与优势

- **统一改写上下文**：用户不需要关心不同来源的内容如何合并，只需把相关材料逐条加入队列，系统会在后端按顺序聚合为一个统一上下文，让模型能够在同一轮中理解和改写。
- **扩展性好**：新增一种输入来源时（例如小程序、微信公众号文章等），只要在队列结构中增加新的 `type`，在后端抽取阶段实现对应的抽取器，即可无缝接入，不影响现有改写流程。
- **可观测性与容错**：抽取流程中对每一步都记录日志；单个 URL/文件失败不会直接导致整次请求失败（视场景决定是跳过还是抛错），并通过自定义异常与统一错误响应体系，给前端提供稳定的反馈。

通过多源输入队列处理机制，ReAngle 能够自然支持「多篇文章合并改写」「文档 + 网页 + 自己笔记组合」等实际使用场景，为后续更多内容形态的接入（如小程序、视频字幕等）提供了清晰的扩展路径。
