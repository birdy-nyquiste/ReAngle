# 多语言功能实现

本文档说明 ReAngle 前端如何实现多语言（当前支持英文、中文、西班牙语）：从文案管理、语言状态、统一头部到各页面接入的完整流程。用户可在任意页面通过头部下拉切换语言，选择会写入 `localStorage` 并在下次访问时沿用；若未保存过偏好，则根据浏览器语言做初始猜测。

---

## 第一步：文案与翻译查找（locales）

**目标：** 把界面上的所有文案集中到「按 key 取文案」的结构里，并提供一个通用的查找函数，供后续 Context 里的 `t(key)` 使用。

### 1.1 新建 locale 文件

在 `frontend/src/locales/` 下建立三种语言的文案文件，结构一致、按命名空间分组：

| 文件   | 说明           |
|--------|----------------|
| `en.ts` | 英文（主语言，定义结构） |
| `zh.ts` | 中文           |
| `es.ts` | 西班牙语       |

每个文件导出一个大对象，内部按功能分块，例如：`nav`（导航）、`landing`（落地页）、`mainApp`（主应用）、`auth`（登录/注册）、`pricing`（定价）、`profile`（个人中心）、`common`（通用）等。同一 key 在不同语言文件中一一对应，例如 `nav.pricing` 在英文里是 `"Pricing"`，在中文里是 `"定价"`。

### 1.2 统一导出与查找函数

在 `frontend/src/locales/index.ts` 中：

- 导出 `en`、`zh`、`es` 以及英文的类型 `LocaleEn`（供 `zh`、`es` 参考结构）。
- 实现 **`getMessage(messages, key)`**：  
  - `key` 使用点号路径，如 `"nav.pricing"`。  
  - 从传入的 `messages` 对象中按路径取值；若最终得到字符串则返回，否则返回 `key` 本身（便于发现缺译）。

这样，后续只要传入「当前语言的 messages」和 key，就能拿到对应语言的文案。

---

## 第二步：语言状态与 Context

**目标：** 在应用内维护「当前语言」状态，提供切换函数和按 key 取文案的 `t(key)`，并做持久化与初始化。

### 2.1 创建 LanguageContext

在 `frontend/src/context/LanguageContext.tsx` 中：

- **语言类型：** `Language = "en" | "zh" | "es"`。
- **初始语言逻辑 `getInitialLanguage()`：**  
  1. 若在服务端渲染（无 `window`），返回 `"en"`。  
  2. 读 `localStorage` 的键 `reangle-lang`；若为 `en`/`zh`/`es` 之一则采用。  
  3. 否则用 `navigator.language` / `navigator.languages[0]` 判断：若匹配中文则 `"zh"`，匹配西班牙语则 `"es"`，否则 `"en"`。
- **Context 提供：**  
  - `language`：当前语言。  
  - `setLanguage(lang)`：切换语言；会写入 `localStorage` 并设置 `document.documentElement.lang`（如 `zh-CN`、`es-ES`、`en`）。  
  - `t(key)`：用当前语言的 messages 调用 `getMessage`，返回对应文案。
- **Provider：** 用 `useState(getInitialLanguage)` 管理 `language`，在 `setLanguage` 和 `useEffect` 中同步更新 `document.documentElement.lang`，便于无障碍与 SEO。

### 2.2 挂载 Provider

在 `frontend/src/App.tsx` 根节点用 **`LanguageProvider`** 包裹整个应用（包在 `AuthProvider` 外或内均可，保证所有需要多语言的页面都在其子树内）。

---

## 第三步：统一头部与语言切换入口

**目标：** 各页不再各自写 header，改为共用 `AppHeader`；在头部 logo 右侧增加语言下拉，并随路由/登录状态切换导航内容。

### 3.1 创建 AppHeader 组件

在 `frontend/src/components/AppHeader.tsx` 中：

- 使用 `useLocation()`、`useAuth()` 判断当前是否落地页、主应用、登录/注册、定价、个人中心等，从而决定 header 的样式类（如透明/深色）和右侧导航项（如「登录 / 注册」「定价」「进入应用」「个人中心 / 退出」等）。
- **语言下拉：** 使用 shadcn 的 `Select`，当前值显示为「Language: EN」「语言：中文」或「Idioma: ES」；下拉列表为三种语言，当前选中的项可设为 `disabled`。  
  `value` 绑定 Context 的 `language`，`onValueChange` 调用 `setLanguage`。
- 点击 logo 跳转首页（`navigate("/")`）。

### 3.2 各页用 AppHeader 替换原有 header

从以下页面中移除本地的 `<header>...</header>` 或等效顶部区域，改为只渲染 **`<AppHeader />`**：

- `LandingPage.tsx`
- `MainApp.tsx`
- `LoginPage.tsx`
- `RegisterPage.tsx`
- `PricingPage.tsx`
- `ProfilePage.tsx`

这样所有页面的顶部与语言切换入口一致，且只需在一处维护。

---

## 第四步：页面文案改为 t(key)

**目标：** 把所有页面和共用组件里原先写死的英文（或中文）文案，改成用 `useLanguage()` 取到的 `t(key)` 渲染，使界面随当前语言切换。

### 4.1 使用方式

在需要多语言的组件中：

```ts
import { useLanguage } from "@/context/LanguageContext"

// 在组件内：
const { t } = useLanguage()
// 文案处：t("nav.pricing")、t("landing.heroTitleLine1") 等
```

保证该组件位于 `LanguageProvider` 子树内（已由 App 根节点包裹，无需额外操作）。

### 4.2 已接入的页面与组件

| 页面/组件 | 主要 namespace | 说明 |
|-----------|----------------|------|
| `LandingPage.tsx` | `landing`、`nav` | 英雄区、洞察、叙事引擎、使用步骤、适用人群、底部 CTA、页脚等 |
| `MainApp.tsx` | `mainApp`、`nav` | 输入源、配置项、结果区、错误与按钮文案等 |
| `LoginPage.tsx` | `auth`、`common` | 标题、表单、按钮、链接 |
| `RegisterPage.tsx` | `auth`、`common` | 注册表单、成功态、错误信息 |
| `PricingPage.tsx` | `pricing`、`common` | 方案名称、特性列表、按钮与错误提示 |
| `ProfilePage.tsx` | `profile`、`nav` | 卡片标题、计划状态、用量、订阅与按钮 |
| `ProtectedRoute.tsx` | `common` | 加载中的「Loading...」等 |

新增或修改界面文案时，在 `en.ts`（及 `zh.ts`、`es.ts`）中增补对应 key，再在组件中用 `t("命名空间.键名")` 引用。

---

## 第五步：扩展新语言（以西班牙语为例）

若要在现有中英基础上增加一种语言（如西班牙语），按以下做即可：

1. **文案：** 在 `locales/` 下新增 `es.ts`，结构与 `en.ts` 一致，键与 `en` 一一对应，值为西班牙语文案。
2. **导出与查找：** 在 `locales/index.ts` 中导出 `es`；Context 中已有按 key 从某语言对象取值的逻辑，无需改 `getMessage`。
3. **类型与状态：** 在 `LanguageContext.tsx` 中把 `Language` 改为 `"en" | "zh" | "es"`，在 `getInitialLanguage()` 中增加对浏览器西班牙语的判断（如 `navigator.language` 匹配 `^es`），并把 `messages` 与 `setLanguage`/`document.documentElement.lang` 的支持扩展到 `"es"`（如设为 `es-ES`）。
4. **头部：** 在 `AppHeader.tsx` 中把语言列表改为 `["en", "zh", "es"]`，并为当前语言显示「Idioma: ES」、下拉项「Español」等。

其他语言（如法语、日文）可同理增加：新 locale 文件 → 导出 → Context 与 AppHeader 中枚举新语言。

---

## 涉及文件一览

| 路径 | 作用 |
|------|------|
| `frontend/src/locales/en.ts` | 英文文案（主结构） |
| `frontend/src/locales/zh.ts` | 中文文案 |
| `frontend/src/locales/es.ts` | 西班牙语文案 |
| `frontend/src/locales/index.ts` | 导出 locale 与 `getMessage` |
| `frontend/src/context/LanguageContext.tsx` | 语言状态、`t(key)`、初始化与持久化 |
| `frontend/src/App.tsx` | 根节点挂载 `LanguageProvider` |
| `frontend/src/components/AppHeader.tsx` | 统一头部与语言下拉 |
| 各页面与 `ProtectedRoute.tsx` | 使用 `useLanguage().t(key)` 替代硬编码文案 |

按上述步骤实现后，即可在整站范围内通过头部下拉切换英文、中文、西班牙语，并在刷新或再次访问时保持用户上次选择的语言。
