# 文档覆盖说明（Overview / Architecture / Features / Diagrams / SQL）

本文档说明 ReAngle 项目文档的五个分支及其内容与作用，便于读者快速定位所需文档。

---

## 一、结构总览

| 分类 | 内容 | 作用 |
|------|------|------|
| **Overview** | 项目总览、New Value Proposition、DeAngle功能定义.pdf、用户使用流程 | 项目是什么、解决什么问题、价值主张与产品定义；用户从 Gather Input 到出结果、看 Summary、下载改写正文、生成数字人的完整使用步骤。 |
| **Architecture** | 技术设计、开发环境搭建、用户系统技术文档、错误处理与日志系统 | 系统怎么搭、怎么跑、认证/计费/错误怎么实现。 |
| **Features** | 多源输入、DeAngle、改写与摘要、多模型配置、结果展示、数字人播报、多语言、账号与计费、小程序对接 | 每个功能做什么、用户怎么用、接口与数据长什么样。 |
| **Diagrams** | context、container、component、sequence_core、sequence_subscription 等 | 架构与流程的图示（系统边界、组件、时序等）。 |
| **SQL** | create_user_settings、ensure_user_settings、ensure_tts_usage_controls、ensure_avatar_usage_controls 等 | 数据库脚本（用户设置表、用量控制等），供实施与运维使用。 |

---

## 二、各分类文档清单

### Overview（`docs/overview/`）

- **项目总览**：价值主张、用户流程概要、功能矩阵、技术协作方式与后续方向，并链到各子文档。
- **New Value Proposition**：理念、三阶段 Workflow、目标用户。
- **DeAngle功能定义.pdf**：DeAngle 的详细产品定义与流程。
- **用户使用流程**：从 Gather Input 的输入类型（Text / File / URL / YouTube）、第二步 DeAngle 与 ReAngle、到结果区看 Summary（查看与朗读）、改写正文下载、Compare 对照、数字人口播稿与视频生成及下载，一条文档串起整条使用路径。

### Architecture（`docs/architecture/`）

- **技术设计**：架构、v2 路由、主链路（Inputs → DeAngle → ReAngle）、订阅、Avatar、横切能力。
- **开发环境搭建**：克隆、.env、启动、常见问题、HeyGen 等配置。
- **用户系统技术文档**：Supabase 认证、用量、Stripe、支付端点、表结构、文件索引。
- **错误处理与日志系统**：loguru、request_id、异常类型、API 错误响应。

### Features（`docs/features/`）

| 项目能力 | 对应文档 |
|----------|----------|
| 多源输入入队与抽取 | 多源输入队列处理.md |
| 事件与视角拆解 + 事实核查 | DeAngle功能说明.md |
| 改写与摘要（ReAngle） | 改写与摘要（ReAngle）.md |
| 模型与系统 Prompt 配置 | 多模型调用配置.md |
| 结果视图（Summary/Rewritten/Compare/Avatar）、TTS | 结果展示与交互视图.md |
| 数字人播报 | 数字人播报.md |
| 多语言 | 多语言功能实现.md |
| 账号与订阅计费 | 账号登录与订阅计费.md |
| 小程序对接（独立 API） | 小程序对接.md |

### Diagrams（`docs/diagrams/`）

- **context**：系统边界与外部依赖。
- **container**：容器级结构。
- **component**：组件级结构。
- **sequence_core**：主链路时序（Inputs → DeAngle → ReAngle）。
- **sequence_subscription**：订阅计费时序。

### SQL（`docs/sql/`）

- **create_user_settings**、**ensure_user_settings**：用户设置表及存在性检查。
- **ensure_tts_usage_controls**、**ensure_avatar_usage_controls**：TTS 与数字人用量控制相关数据库对象。

## 三、文档体系说明

上述五类文档共同构成 ReAngle 项目文档体系：Overview 回答项目与价值、用户怎么用；Architecture 回答技术实现与环境；Features 回答各功能与接口；Diagrams 以图示呈现系统与流程；SQL 提供可执行的数据库脚本，与用户系统及用量逻辑配合使用。
