# ReAngle 开发环境搭建

按以下四步完成克隆、配置、依赖安装与启动。前置要求：本机已安装 **Python 3.9+**、**Node.js 20+**、**Git**。

---

## 第一步：克隆项目

```bash
git clone https://github.com/birdy-nyquiste/ReAngle.git
cd ReAngle
```

之后所称「项目根目录」即 `ReAngle` 目录（Windows 下例如 `.\ReAngle`）。

---

## 第二步：配置两个 .env 文件

在本地新建两个 `.env` 文件，**不要**提交到 Git（项目已忽略 `.env`）。

### 1. 根目录 `.env`（后端用）

路径：`ReAngle\.env`（与 `requirements.txt` 同级）。

可参考 README 中的环境变量模板，建议包含以下内容：

| 配置项 | 说明 | 必要性 |
|--------|------|--------|
| `SUPABASE_URL` | Supabase 项目 URL | **必填**：登录与鉴权依赖 |
| `SUPABASE_PUBLISHABLE_KEY` | Supabase 匿名公钥 (anon key) | **必填** |
| `SUPABASE_SECRET_KEY` | Supabase 服务端密钥 (service_role key) | **必填**：后端校验 JWT |
| `OPENAI_API_KEY` / `GEMINI_API_KEY` / `DASHSCOPE_API_KEY` | 任选其一或多项，用于文章改写与摘要 | **至少填一个**，否则改写功能不可用 |
| `FRONTEND_URL` | 前端地址，用于 CORS | **可选**，默认 `http://localhost:5173` |
| `STRIPE_PUBLISHABLE_KEY`、`STRIPE_SECRET_KEY`、`STRIPE_WEBHOOK_SECRET`、`PRICE_ID_PRO` | Stripe 订阅与支付 | **可选**，仅在使用付费/订阅时配置 |
| `HEYGEN_API_KEY` | HeyGen 数字人视频 API Key | **可选**（仅在使用 Avatar 播报时需要） |
| `API_URL` | 后端 API 地址 | **可选**，当前后端未使用 |

**示例（占位，勿填真实密钥进文档）：**

```env
# Supabase（必填）
SUPABASE_URL=your-supabase-url
SUPABASE_PUBLISHABLE_KEY=your-supabase-anon-key
SUPABASE_SECRET_KEY=your-supabase-service-role-key

# LLM（至少填一个）
OPENAI_API_KEY=your-openai-key

# 应用 URL（可选，本地可不写）
FRONTEND_URL=http://localhost:5173
```

如需在本地联调 Avatar 数字人播报功能，至少需要在根目录 `.env` 中配置 `HEYGEN_API_KEY`，并根据需要增加可选参数，例如：

- `HEYGEN_AVATAR_ID`、`HEYGEN_VOICE_ID`：指定数字人形象和声音；
- `HEYGEN_VIDEO_WIDTH`、`HEYGEN_VIDEO_HEIGHT`：控制视频分辨率（如 `640x360`）；
- `HEYGEN_BACKGROUND_COLOR` 等：控制背景颜色与填充方式。

若未配置这些可选项，后端的 `AvatarClient` 会使用内置的默认值，仍能正常调用 HeyGen 生成数字人视频。

### 2. 前端目录 `frontend\.env`

路径：`ReAngle\frontend\.env`。前端仅读取以 `VITE_` 开头的变量。

| 配置项 | 说明 | 必要性 |
|--------|------|--------|
| `VITE_SUPABASE_URL` | 与根目录 `SUPABASE_URL` 一致 | **必填**，缺则可能白屏/报错 |
| `VITE_SUPABASE_PUBLISHABLE_KEY` | 与根目录 `SUPABASE_PUBLISHABLE_KEY` 一致 | **必填** |
| `VITE_STRIPE_PUBLISHABLE_KEY` | Stripe 公钥，用于前端跳转支付页 | **可选** |
| `VITE_API_URL` | 前端直连的后端地址 | **可选**，本地开发走 Vite 代理，一般不需要 |

**示例：**

```env
VITE_SUPABASE_URL=your-supabase-url
VITE_SUPABASE_PUBLISHABLE_KEY=your-supabase-anon-key
```

## 第三步：两个终端分别启动后端与前端

**顺序：先启动后端，再启动前端。**

### 终端一：后端

- 当前目录：**项目根目录** `.\ReAngle`（若使用虚拟环境，先激活再执行下面命令）。

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

- 后端默认运行在 **http://localhost:8000**。

### 终端二：前端

- 当前目录：**前端目录** `.\ReAngle\frontend`（从项目根目录执行 `cd frontend` 即可）。

```bash
cd frontend
npm install
npm run dev
```

- 前端默认运行在 **http://localhost:5173**。开发环境下，前端的 `/api` 请求会通过 Vite 代理到 `http://127.0.0.1:8000`。

---

## 第四步：访问项目

浏览器打开 **http://localhost:5173** 即可查看并使用 ReAngle 前端；登录与改写功能依赖第二步中配置的 Supabase 与至少一个 LLM Key。

---

## 常见问题速查

- **401 / session expired / `supabase_url is required`**：检查根目录 `.env` 中 Supabase 三项是否填写正确，修改后需重启后端（重新执行 `uvicorn app.main:app --reload`）。
- **前端白屏或报错缺少 Supabase 变量**：检查 `frontend\.env` 中 `VITE_SUPABASE_URL` 与 `VITE_SUPABASE_PUBLISHABLE_KEY`，修改后需重新执行 `npm run dev`。
- **改写报错或提示未配置 API Key**：在根目录 `.env` 中至少配置 `OPENAI_API_KEY`、`GEMINI_API_KEY` 或 `DASHSCOPE_API_KEY` 其中之一。

更多说明见根目录 [README.md](../README.md)，其他技术文档见 [用户系统技术文档](../architecture/用户系统技术文档.md)、[错误处理与日志系统](../architecture/错误处理与日志系统.md) 等。
