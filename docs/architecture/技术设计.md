# Technical Design (ReAngle)

本技术设计说明 ReAngle 的系统架构、后端与前端模块分层、关键链路（改写/订阅/Avatar/小程序）及横切能力（错误处理、安全）；具体实现见各功能与架构文档。

---

## 1. System Architecture（系统架构总览）

ReAngle 采用典型的“Web 前端 + 后端 API + BaaS + 第三方 AI/支付”的架构：

- **Frontend（React + Vite + TypeScript）**
  - 负责 UI、输入队列、结果视图（Summary / Rewritten / Compare / Avatar）与交互编排。
  - 通过 Supabase SDK 维护登录会话，并在调用后端时携带 `Bearer token`。
- **Backend（FastAPI）**
  - 所有 API 挂载在 **`/api/v2`** 下：多源输入（inputs）、DeAngle、ReAngle、媒体（TTS/口播稿/数字人/下载）、支付、设置等。
  - 统一鉴权、用量控制、错误处理与日志，再分发到各服务模块。
- **Supabase（Auth + Postgres）**
  - 负责账号认证（JWT）、用户资料与用量表、订阅状态存储、原子用量递增 RPC。
- **Stripe（Checkout + Webhook + Portal）**
  - 负责订阅计费生命周期，后端通过 Webhook 同步订阅状态与权益。
- **External Providers**
  - **LLM Providers**：OpenAI / Gemini / Qwen（文章改写与摘要）
  - **DashScope**：TTS（朗读摘要）
  - **HeyGen**：Avatar 视频（数字人播报）

## 2. Backend 模块分层与职责

后端 FastAPI 按“路由层 + 核心基础设施 + 应用服务 + Provider Clients”的方式组织：

1. **Router Layer（app/routers/v2/）**
   - **inputs**：`POST /api/v2/inputs/`，多源输入队列抽取与合并，结果写入 Session。
   - **deangle**：`POST /api/v2/deangle/`，对 Session 内原文做 Event/Angle 拆分与事实核查。
   - **reangle**：`POST /api/v2/reangle/`，基于 Session 与用户选择的 facts/angles 及 prompt 生成摘要与改写正文。
   - **media**：`POST /api/v2/media/tts`、`/voiceover`、`/avatar`，`GET /api/v2/media/avatar/download`（TTS、口播稿、数字人视频及下载代理）。
   - **payment**：订阅计费（Checkout/Portal/Webhook/Usage）。
   - **settings**：用户设置（如 ReAngle 模型、系统 Prompt 等）。

2. **Core Infrastructure（core/）**
   - 鉴权（JWT）、用量检查与递增、统一异常与 API 错误响应、request_id 与请求/响应日志。

3. **Inputs / ReAngle 应用服务**
   - Inputs 服务：多源内容抽取（URL/PDF/DOCX/图片等）并合并为 `clean_text` 存入 Session。
   - ReAngle 服务：基于 Session 中的原文与 DeAngle 结果、用户 prompt 与所选模型，调用 LLM 生成 `summary` 与 `rewritten_content`。

4. **Provider Clients（第三方能力封装）**
   - LLM（OpenAI / Gemini / Qwen，由用户设置中的 reangle_model 等驱动）。
   - TTS（DashScope）。
   - Avatar（HeyGen **Video Generation API**：`/v2/video/generate`，由 `app/services/media_outputs/avatar_client.py` 封装，无 Video Agent）。

## 3. Frontend 关键模块与交互编排

前端的核心交互集中在 `MainApp` 页面，采用“左侧输入/配置 + 右侧结果面板”的布局：

- **输入队列（Multi-source Input Queue）**
  - 支持 Text / File / URL 等多源输入，逐条入队后提交到 `POST /api/v2/inputs/`，结果存入 Session；可选执行 DeAngle 再执行 ReAngle。
- **模型与设置（Settings）**
  - 用户可在设置页选择 ReAngle 所用模型（如 ChatGPT / Gemini / Qwen）及系统 Prompt，ReAngle 请求时由后端从用户设置读取。
- **结果视图（Tabs）**
  - Summary：摘要展示 + TTS 播放
  - Rewritten：改写正文 + 下载
  - Compare：原文 vs 改写 diff
  - Avatar：口播稿生成 + 编辑 + 数字人视频生成 + 下载

## 4. 关键链路（End-to-End Flows）

### 4.1 主链路（Inputs → DeAngle → ReAngle → 结果）

主路径：用户将多源材料入队并提交 **`POST /api/v2/inputs/`**，后端抽取并合并为 `clean_text` 写入 Session；可选 **`POST /api/v2/deangle/`** 做 Event/Angle 拆分与事实核查；再 **`POST /api/v2/reangle/`** 根据用户选择的 facts/angles 与 prompt、以及用户设置中的模型，调用 LLM 生成 `summary` 与 `rewritten_content`。

后端关键点：鉴权（JWT）、用量控制（Inputs/ReAngle 等消耗配额，超限 402）、多源抽取（inputs 服务）、ReAngle 按用户设置中的模型调用对应 LLM。

可视化参考：`docs/diagrams/sequence_core.md`

### 4.2 Subscription/Billing 链路（升级 → Checkout → Webhook 同步 → 权益生效）

订阅计费以 Stripe 为权威数据源，后端采用 **State-Based Sync**：Webhook 只作为“变更通知”，收到后主动查询 Stripe 的订阅权威状态并落库到 Supabase，更新用户权益与额度。

可视化参考：
- `docs/diagrams/sequence_subscription.md`
- 详细技术说明：`docs/architecture/用户系统技术文档.md`
- 功能说明：`docs/features/账号登录与订阅计费.md`

### 4.3 Avatar（口播稿 → 数字人视频 → 预览/下载）

Avatar 建立在 ReAngle 结果之上：前端将改写正文发往 **`POST /api/v2/media/voiceover`** 生成口播稿，再以口播稿调用 **`POST /api/v2/media/avatar`**；后端仅使用 HeyGen **Video Generation API**（`/v2/video/generate`），通过环境变量配置 avatar、声音、分辨率与背景，口播稿作为 `input_text` 传入。下载通过 **`GET /api/v2/media/avatar/download?url=...`** 代理。前端对口播稿长度做上限（如 800 字）以控制成本与时长。

详细说明：`docs/features/数字人播报.md`

### 4.4 小程序对接（可选/独立）

小程序对接文档描述独立的小程序侧 API 使用方式（健康检查、生成故事、获取结果等），当前主后端为 v2 API；若存在独立小程序服务，可参考 `docs/features/小程序对接.md`。

## 5. Cross-Cutting Concerns（横切能力）

### 5.1 Error Handling & Logging

系统使用 `loguru` 做统一日志，配合 request_id 贯穿请求生命周期；所有 API 错误通过统一异常类型和全局 handler 输出标准 JSON 结构，便于前端展示与后端排查。

详细说明：
- `docs/architecture/错误处理与日志系统.md`

### 5.2 Security & Keys

- 前端只持有 Supabase `anon key` 与 Stripe publishable key（如配置）。
- 后端持有 Supabase `service_role key` 与 Stripe secret key，并负责所有敏感操作（JWT 校验、Webhook 验签、权益同步）。
- 对外下载（如 HeyGen 视频下载）通过后端代理并做域名白名单校验，避免 SSRF。

从技术能力维度，当前系统的主要特性包括：

- 多源输入队列抽取与聚合（Text/File/URL/YouTube）
- 多模型路由（ChatGPT / Gemini / Qwen）
- 结果视图（Summary/Rewritten/Compare/Avatar）
- 语音朗读（TTS）
- 数字人播报（口播稿生成 + HeyGen 视频生成 + 下载代理）
- 登录与订阅计费（Supabase Auth + Stripe Subscription + 用量控制）
- 统一错误处理与日志体系（可观测性）
- 小程序对接文档见 `docs/features/小程序对接.md`（独立/历史对接说明）
