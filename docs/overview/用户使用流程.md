# 用户使用流程（完整）

本文档从**用户视角**描述 ReAngle 的完整使用流程：从 Gather Input（收集输入）到出结果、看 Summary、下载改写正文、生成数字人播报，形成一条连贯的操作路径。各步骤的接口与实现细节见对应 Features 文档。

---

## 一、第一步：Gather Input（收集输入）

用户在主应用左侧**输入区**将待处理的素材加入队列。

### 1.1 输入方式有哪几种

当前支持以下**多种输入类型**，可单独或组合使用：

| 类型 | 说明 | 操作方式 |
|------|------|----------|
| **Text（纯文本）** | 直接粘贴或输入一段文字 | 在「Text」Tab 中输入后点击「Add to Queue」 |
| **File（文件）** | 上传文档，系统抽取正文 | 支持 `.docx`、`.pdf` 等，在「File」Tab 选择文件后加入队列 |
| **URL（网页链接）** | 由系统抓取该页正文 | 在「URL」Tab 粘贴链接后加入队列 |
| **YouTube** | 视频链接（当前以链接文本纳入上下文，可扩展为字幕） | 在对应 Tab 粘贴链接后加入队列 |

每次在对应 Tab 填好内容后点击「Add to Queue」，该条会作为**一条记录**出现在队列列表中（可看到标题/文件名、字符数等简要信息）。可多次添加、多种类型混合，也可从队列中删除某条。  
队列准备好后，用户点击提交（如「Transform Content」或当前流程的提交按钮），前端将整份队列提交给后端 **`POST /api/v2/inputs/`**，后端统一抽取并合并为一份原文（`clean_text`）写入 Session，供后续步骤使用。

**对应文档**：`docs/features/多源输入队列处理.md`

---

## 二、第二步：DeAngle（可选）与 ReAngle（改写与摘要）

- **DeAngle（可选）**  
  用户可执行 **DeAngle**：系统对第一步产出的原文做「事件（Event）与视角（Angle）」拆分，并对事件做事实核查，结果以事件列表、视角列表的形式展示；用户可在前端勾选要采用的事件与视角。  
  对应接口：`POST /api/v2/deangle/`。详见 `docs/features/DeAngle功能说明.md`。

- **ReAngle（改写与摘要）**  
  用户在前端填写**新叙事角度 / 风格**（prompt），并基于 DeAngle 结果勾选要采用的事实与视角（若未做 DeAngle 则按产品逻辑处理）。提交后，后端根据用户设置中的模型与系统 Prompt 生成：  
  - **summary**：摘要  
  - **rewritten_content**：改写后的正文  

  对应接口：`POST /api/v2/reangle/`。详见 `docs/features/改写与摘要（ReAngle）.md`。

完成 ReAngle 后，结果会在右侧**结果区域**以多个标签页呈现。

## 三、第三步：查看结果与后续操作

结果区包含四个主要标签，用户可依需切换。

### 3.1 Summary（摘要）

- **做什么**：展示 ReAngle 返回的**摘要**，便于快速把握要点。  
- **可做操作**：  
  - **查看**：在页面直接阅读摘要。  
  - **朗读**：点击「朗读」可生成并播放 TTS 音频（调用 `POST /api/v2/media/tts`）；适合听摘要。  
- **说明**：摘要以页面展示与朗读为主；若需留存，可复制页面文本。改写正文的下载见下条。

### 3.2 Rewritten（改写正文）

- **做什么**：展示完整的**改写后文章**（`rewritten_content`），是复制、发布和下载的主要来源。  
- **可做操作**：  
  - **查看**：在结果区滚动阅读全文。  
  - **下载**：点击「Download」可将改写正文打包为 **.txt** 文件并触发浏览器下载，无需再调后端。

### 3.3 Compare（对照）

- **做什么**：原文与改写稿**并排对照**，差异高亮，便于检查信息是否保留、语气是否符合预期。  
- **可做操作**：仅查看与对比，无单独下载；改写正文的下载在 Rewritten 标签完成。

### 3.4 Avatar（数字人播报）

- **做什么**：在改写结果基础上，生成**口播稿**并进一步生成**数字人视频**，支持预览与下载。  
- **可做操作**：  
  1. **生成口播稿**：基于当前改写正文调用 `POST /api/v2/media/voiceover`，得到适合朗读的口播稿，可在文本框内编辑（如删减、润色）。  
  2. **生成数字人播报**：以当前口播稿调用 `POST /api/v2/media/avatar`，生成数字人视频；若尚未生成口播稿，系统会先自动生成再生成视频。口播稿过长（如超过约 800 字）时前端会提示先缩短。  
  3. **预览与下载视频**：生成完成后在页面内嵌播放器预览；点击「下载视频」通过 `GET /api/v2/media/avatar/download?url=...` 代理下载视频文件。

**对应文档**：`docs/features/结果展示与交互视图.md`、`docs/features/数字人播报.md`

## 四、流程串联小结

| 步骤 | 用户动作 | 结果/产出 |
|------|----------|------------|
| 1. Gather Input | 用 Text / File / URL / YouTube 入队并提交 | 原文合并入 Session |
| 2a. DeAngle（可选） | 执行 DeAngle，勾选事件与视角 | 事件列表、视角列表写入 Session |
| 2b. ReAngle | 填写 prompt，提交 ReAngle | summary + rewritten_content |
| 3. 看结果 | Summary：查看、朗读；Rewritten：查看、**下载 .txt**；Compare：对照；Avatar：口播稿 → 数字人视频 → **下载视频** | 阅读、下载、播报与留存 |

上述流程在**项目总览**中有概要描述；各步的接口、参数与实现见 **Features** 下对应文档。
